# HabitSprout 设计工作流与设计哲学 (v2.0 - 精灵养成版)

本文档概述了 **HabitSprout (习惯小芽)** 的构思、设计和实现工作流。它旨在解释设计决策背后的“原因”，并为未来的迭代提供蓝图。

## 1. 产品愿景与核心哲学

### 问题痛点
传统的习惯追踪器往往让人感到僵硬、机械化且过于复杂。不同的 App 可能针对不同的功能适配不同的人群，且打卡模式比较冰冷，没有趣味，让人无法坚持。这些 App 的打卡功能往往只是附属品，设计得不够有灵性，且其它内容容易分散人们注意力。

### 解决方案：HabitSprout (精灵版)
HabitSprout 定位为 **“治愈系精灵伴侣”**。
*   **核心隐喻：** **养成/羁绊**。你的自律是精灵成长的养分。你坚持打卡，精灵就会长大。
*   **情感基调：** 治愈、温柔、陪伴感、轻游戏化。
*   **价值主张：** “你的每一次坚持，都让它变得更好。”

---

## 2. 视觉识别系统 (The Vibe)

保持初版备受好评的 **“有机生活方式”** 美学，将二次元/游戏元素完美融入其中，避免突兀的“电玩感”。

### A. 色彩与材质 (保持不变)
*   **画布:** `bg-green-50` (#f0fdf4) 配合噪点纹理，营造纸张质感。
*   **UI:** 毛玻璃卡片 (`backdrop-filter`)，营造轻盈、悬浮的高级感。
*   **融合:** 精灵角色将以“立绘”或“Q版形象”的形式，自然地站在毛玻璃卡片之上，仿佛打破了次元壁。

---

## 3. 核心功能设计：精灵养成打卡 (Elf Habit Tracker)

这是产品的灵魂功能。我们将“打卡”这一枯燥动作，转化为“培育”的乐趣。

### A. 角色设定 (The Companion)
*   **多样性:** 精灵可以是原创的自然之灵（如芽芽、露珠），也可以是用户喜欢的动漫/游戏角色（Q版化）。
*   **状态:** 精灵始终显示在打卡页面的最上方（Hero 区域）。

### B. 成长机制 (Leveling System)
*   **等级上限:** **Lv.21** (对应 21 天习惯养成效应)。
*   **经验值 (XP):**
    *   **来源:** 每次完成一个习惯打卡。
    *   **反馈:** 点击打卡 -> 精灵头顶浮现 `XP +1` 或 `XP ↑` 动画 -> 经验条增长。
*   **阶段变化:**
    *   **Lv.1 - Lv.7 (萌芽期):** 初始形态（如：一颗蛋、一个小幼苗、一个趴着的Q版人物）。
    *   **Lv.8 - Lv.14 (成长期):** 形态发生变化，站立起来，表情更丰富。
    *   **Lv.15 - Lv.21 (成熟期):** 最终形态，拥有华丽的特效或专属背景。

### C. 交互流程 (User Flow)
1.  **进入页面:** 看到精灵在页面上方（根据当前状态，可能是发呆、看书等）。
2.  **执行打卡:** 用户点击下方的习惯列表（如“喝水”）。
3.  **即时反馈:**
    *   习惯项变绿（已完成）。
    *   **视觉焦点转移:** 精灵头顶冒出经验值气泡。
    *   **进度条:** 顶部的等级进度条向前推进。
4.  **升级时刻:** 当达到升级阈值（如累计打卡满 1 天、3 天、7 天），触发升级动画（光效、撒花），精灵形态改变。

## 4. 技术实现策略 (更新版)

### 数据结构 (localStorage)
我们需要扩充数据模型来支持养成系统：

```json
{
  "user": {
    "level": 1,
    "currentXP": 0,
    "maxXP": 100, // 随等级增加
    "characterId": "char_001" // 当前选择的精灵
  },
  "habits": [...],
  "records": { ... }
}
```

### UI 布局调整
*   **Header 区域:** 改造为“精灵展示区”。
    *   背景：保留真实照片（模糊处理）。
    *   前景：精灵图片（居中）。
    *   覆盖层：等级徽章 (Lv.5) 和 经验条 (Progress Bar)。
*   **列表区域:** 保持现有的简洁设计，确保打卡操作顺滑。

## 5. 未来路线图 (Next Steps)

1.  **精灵图鉴:** 允许用户解锁和切换不同的精灵角色。
2.  **互动对话:** 点击精灵，它会根据你的打卡情况说不同的台词（“今天也要加油哦！” / “哇，你真棒！”）。
3.  **多阶段形态:** 为每个角色设计 3 种不同的成长形态图片。
